# PRD: Permission-Aware Enterprise GraphRAG Assistant

## 1. 产品定位

Permission-Aware Enterprise GraphRAG Assistant 是一个面向企业内部知识库问答场景的权限感知AI应用样板。它强调企业级权限边界、审计、缓存安全和RAG/GraphRAG检索链路，而不是只做聊天界面。

中文名：企业级权限感知GraphRAG知识问答系统。

## 2. 目标用户

- 招聘方或技术面试官：评估候选人是否具备企业AI应用架构和落地能力。
- 企业内部员工：按角色和部门访问授权知识库。
- 管理员：管理用户、部门、知识库和审计日志。

## 3. 核心角色

| 角色 | 描述 |
| --- | --- |
| `admin` | 管理员，可管理系统资源和查看审计日志 |
| `hr` | HR部门用户，只能访问HR知识库和公开知识库 |
| `finance` | 财务部门用户，只能访问财务知识库和公开知识库 |
| `tech` | 技术部门用户，只能访问技术知识库和公开知识库 |
| `visitor` | 访客，只能访问公开知识库 |

## 4. MVP目标

MVP必须能够展示：

- 用户可以登录。
- 后端能根据JWT识别用户身份。
- 系统能根据角色、部门、知识库ACL计算访问范围。
- 用户提问后，后端先做权限判断，再做检索。
- 未授权知识库不会进入RAG召回结果。
- 访客询问财务薪酬制度时被拒绝。
- Redis缓存key不会导致不同权限用户共享答案。
- 问答请求和拒绝请求都写入审计日志。
- 本地轻量模型路由接口被预留，MVP可用规则模拟。
- 高参数模型通过OpenAI-compatible API接口调用。

## 5. 核心用户故事

- 作为 `visitor`，我只能查看公开知识库，不能读取HR、财务、技术内部制度。
- 作为 `hr`，我希望查询HR制度，并获得带引用的答案。
- 作为 `finance`，我希望查询财务报销和预算制度。
- 作为 `tech`，我希望查询技术运维、发布和安全规范。
- 作为 `admin`，我希望查看所有问答审计日志，定位越权尝试。

## 6. 非目标

MVP阶段不追求：

- 真实企业文档接入。
- 复杂多租户组织架构。
- 完整后台管理系统。
- 大规模向量检索性能优化。
- 生产级模型评估平台。
- 真实SSO、SAML或企业LDAP集成。

## 7. 成功标准

- 本地可以启动前后端和依赖服务。
- README能清楚说明架构、权限边界和运行方式。
- 至少提供一组虚构企业示例数据。
- 能演示授权问答和越权拒绝两类流程。
- 审计日志可查。
- 代码中不存在真实API Key、真实企业文档或真实用户数据。
