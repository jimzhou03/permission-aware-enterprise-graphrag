# Robot SDK Manual（tech-internal）

## 1. 适用范围
本文件属于星海智造机器人有限公司 Demo v0.9.4 的虚构技术部内部资料，只允许技术部和管理员访问。技术部 Robot SDK Manual 用于回答机器人 SDK、API 调用、部署排障、日志与遥测相关问题，不对访客开放。

## 2. SDK 接入概览
技术部集成 Robot SDK 时，先确认机器人编号、SDK 版本、API 地址、网络连接、认证参数和测试环境。SDK 客户端初始化后，应完成设备注册、心跳检查、任务调度命令、导航状态读取和遥测订阅。所有步骤都要记录 request_id、robot_id、错误码和耗时。

## 3. API 调用检查
技术部排查 SDK/API 调用异常时，先检查认证范围、请求参数、接口版本、超时、重试策略和返回错误码。若机器人任务创建成功但执行失败，需要继续检查任务队列、路径约束、导航模块、传感器状态和电机状态，不能只看 API 200 响应。

## 4. 部署排障清单
Robot SDK 部署排障清单包括网络连接、DNS、端口、防火墙、设备证书、系统时间、SDK 版本、API 权限、日志上报和遥测可见性。机器人网络连接异常时，技术部先确认网关到 API 的连通性和心跳，再检查代理、丢包率和重试风暴。

## 5. 回归与升级
技术部修复 SDK 或部署问题后，需要执行复现与回归测试：注册、心跳、导航、任务调度、日志上报、传感器告警、电机状态和异常恢复。连续两轮排查仍无法定位，或涉及固件、导航算法、核心安全策略时，升级给高级工程师。
