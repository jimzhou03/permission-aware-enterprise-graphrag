# Deployment Troubleshooting（tech-internal）

## 1. 适用范围
本文件属于星海智造机器人有限公司 Demo v0.9.4 的虚构技术部内部资料，只允许技术部和管理员访问。技术部 Deployment Troubleshooting 用于回答 Robot SDK 部署排障清单、机器人网络连接异常、传感器、电机和导航模块异常检查。

## 2. 部署前检查
技术部部署前检查包括 API 服务健康、数据库和缓存状态、网关时间同步、DNS、端口、防火墙、机器人注册状态、SDK 版本、设备证书和日志上报。未完成部署前检查时，不应进入正式任务调度。

## 3. 网络连接异常
机器人网络连接异常时，技术部先检查网关到 API 的延迟、丢包率、DNS 解析、代理策略和端口连通性，再检查机器人心跳、遥测上报和重试频率。若出现重试风暴，应临时降低轮询频率并启用本地队列缓冲。

## 4. 模块异常检查
传感器、电机、导航模块异常要分别排查。传感器检查健康端点、校准时间、固件版本和环境干扰；电机检查驱动状态、温度、电量和保护告警；导航模块检查地图、禁行区、定位漂移、路径约束和任务队列冲突。

## 5. 复现、回归与升级
技术部部署排障必须记录复现步骤、request_id、robot_id、日志片段和当前结论。修复后执行回归测试，覆盖网络、注册、心跳、任务调度、传感器、电机、导航和日志上报。P1/P2 或连续无法定位的问题升级给高级工程师。

## 6. 可回答的问题
- Robot SDK 部署排障清单是什么？
- 机器人网络连接异常怎么排查？
- 传感器、电机、导航模块异常怎么检查？
- 部署前需要检查哪些环境条件？
- 技术部排障什么时候升级给高级工程师？
