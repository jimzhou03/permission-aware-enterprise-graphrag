# 全员内部通用知识库：组织协作与权限使用规范（Demo v0.7.2）

## 1. 适用范围
本文件仅面向星海智造机器人有限公司正式员工（含跨部门管理员）。访客账号无权访问。

## 2. 文档目的
- 提供全员可共享的内部组织与协作规则。
- 明确跨部门问题应联系的责任部门。
- 统一内部知识库使用与权限申请流程。

## 3. 公司组织架构（演示）
公司采用“职能部门 + 场景项目协同”模式。核心职能部门包括：技术部、销售部、市场部、客服部、人事部、行政部、产品部。日常由部门线条负责专业职能，项目交付由跨部门协作小组推进。

## 4. 各部门职责总览
- 技术部：SDK、API、部署、设备接入、运行排障。
- 销售部：报价策略、客户分级、渠道合作、商务推进。
- 市场部：品牌定位、展会活动、宣传物料与案例口径。
- 客服部：售后流程、工单分级、保修规则、服务升级。
- 人事部：入职、试用期、考勤、绩效、培训制度。
- 行政部：会议室、资产、采购、访客登记、行政审批。
- 产品部：产品规格、版本规划、路线图、竞品分析。

## 5. 跨部门协作流程
1. 发起：提出需求的部门在内部系统创建协作单，写明目标、截止时间、依赖部门。
2. 受理：被协作部门在工作日内确认是否受理并反馈负责人。
3. 评估：多部门评估范围、风险、排期，必要时拉评审会。
4. 执行：按协作单拆解任务并同步状态。
5. 关闭：确认交付结果并沉淀复盘结论。

## 6. 内部知识库使用规范
- 先确认问题所属范围，再进入对应知识库检索。
- 不得将无权限内容复制到对外材料或跨部门公开群。
- 对于未命中信息，应反馈到所属部门补充文档，而不是臆造答案。
- 技术字段（chunk_id、trace 等）仅用于管理员开发者追踪页面。

## 7. 权限申请流程
1. 员工提交权限申请，写明业务场景、所需知识库、期限。
2. 直属主管审批业务必要性。
3. 管理员审批最小权限范围并生效。
4. 到期自动复核；长期不用的权限应回收。

## 8. 常见内部问题分流
- “要做 SDK 接入” -> 技术部。
- “客户要报价/折扣” -> 销售部。
- “需要展会宣传方案” -> 市场部。
- “客户故障与售后升级” -> 客服部。
- “考勤绩效培训规则” -> 人事部。
- “会议室采购资产流程” -> 行政部。
- “产品规格路线图反馈” -> 产品部。

## 9. 内部协作发起指引
- 商务协作：由销售发起，必要时拉产品和技术评估。
- 客户需求转交：销售/市场统一在协作单中转交产品或技术，不走私聊口头流程。
- 售后升级：客服按工单级别触发技术或产品协同。
- 产品反馈：客服、销售、市场将反馈归档到产品需求池。

## 10. 权限边界
- 本文不包含销售底价、技术密钥、客户隐私、工单原始明细。
- 本文用于“协作规则与分流”，不替代各部门内部细则。
- 访客账号不可访问本知识库。

## 11. 可用于问答的明确事实点
- `company-internal` 仅对正式员工和管理员开放，访客不可见。
- 全员问题分流按七大部门职责执行。
- 权限申请流程包括员工申请、主管审批、管理员审批、到期复核。
- 跨部门协作流程是发起、受理、评估、执行、关闭五步。
- 技术 trace 字段只在管理员开发者追踪页面展示。
