# 知识库权限申请指南（company-internal）

## 1. 适用范围
本文件属于星海智造机器人有限公司 Demo v0.9.4 的虚构公司内部权限指南，仅供正式员工和跨部门管理员使用，访客不可访问。本文用于回答“公司内部员工如何申请知识库权限”“怎么申请权限”等稳定演示问题。

## 2. 申请入口
公司内部员工需要新增知识库权限时，应在内部权限申请单中提交申请。申请单必须写明申请人部门、业务场景、目标知识库、预计使用时长、是否涉及跨部门资料，以及申请理由。申请范围应遵循最小权限原则，只申请完成当前工作所需的知识库，不应申请“全部内部资料”。

## 3. 审批流程
权限申请流程为：员工提交申请；直属主管确认业务必要性；知识库管理员核对目标 KB、访问期限和数据安全说明；通过后授予限定范围访问；到期后自动复核或回收。若申请被拒绝，员工应补充业务场景或改为向责任部门发起协作单。

## 4. 访问规则
普通员工只能看到 `public-policy`、`company-internal` 和本部门 `<department>-internal`。访客只能访问 `public-policy`，不能访问公司内部通用知识库。跨部门管理员可按演示需要访问全部 9 个 KB，但管理员 trace 仍只是排查视图，不代表普通员工可见。

- 怎么申请权限？
- 员工能不能查看其他部门资料？
