# Product Production Workflow（product-internal）

## 1. 适用范围
本流程属于星海智造机器人有限公司 Demo v0.9.4 的虚构产品部内部资料，仅适用于产品部内部知识库（`product-internal`），只允许产品部和管理员访问，用于回答“产品生产流程是什么”相关问题。

## 2. 标准流程
1. 需求收集  
从销售、客服、市场、交付与客户反馈中收集需求，并按业务价值初筛。

2. PRD 编写  
输出 PRD，明确业务目标、用户场景、功能边界、验收标准与风险假设。

3. 原型评审  
组织产品、设计、研发、测试联合评审，确认交互路径与异常流程。

4. 技术评审  
评估实现复杂度、依赖组件、数据与权限影响，输出技术方案建议。

5. 生产排期  
确定迭代范围、开发测试窗口、上线闸点与回滚预案。

6. 灰度发布  
先小流量灰度，观察关键指标和故障率，再按计划扩大范围。

7. 反馈复盘  
汇总上线反馈，输出问题清单、改进项与下一版本输入。

## 3. 角色协作
- 产品经理：流程 owner，负责 PRD 与优先级决策。
- 技术负责人：负责技术评估、风险控制与交付可行性。
- 测试负责人：负责质量门禁、回归覆盖与发布验证。

## 4. 权限边界
- 本文属于产品部内部流程，不对访客开放。
- 不包含销售底价、技术密钥、人事档案。
- 产品生产流程只服务产品部和管理员的内部演示，不代表真实企业生产资料。

## 5. 可用于问答的明确事实点
- 产品生产流程固定七步：需求收集、PRD 编写、原型评审、技术评审、生产排期、灰度发布、反馈复盘。
- “产品生产流程”问题应优先命中 `product-internal`。
- “产品部门内部知识库写的什么”可回答：产品需求、PRD、原型评审、技术评审、生产排期、版本发布、复盘和竞品分析。
