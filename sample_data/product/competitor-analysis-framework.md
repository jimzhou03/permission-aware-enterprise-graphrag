# 竞品分析框架（product-internal）

## 1. 适用范围
本文件属于星海智造机器人有限公司 Demo v0.9.4 的虚构产品部内部资料，只允许产品部和管理员访问。本文用于回答“竞品分析框架是什么”等问题，不代表真实市场研究。

## 2. 分析维度
产品部竞品分析框架包括场景覆盖、部署复杂度、任务稳定性、维护成本、权限治理、数据可观测性、售后协同和商业包装八个维度。每个维度只记录演示级事实和内部判断，不包含真实竞品机密或未经授权的客户信息。

## 3. 输出方式
竞品分析输出分为事实表、差异摘要、机会点、风险点和下一步验证计划。事实表记录公开资料可验证内容；差异摘要供产品内部评审使用；机会点进入需求池；风险点进入版本规划或技术预研。不得把内部竞品结论直接用于市场宣传。

## 4. 协作边界
产品部负责竞品框架和需求转化，市场部负责公开传播口径，销售部负责客户沟通，技术部负责可行性评估。若客户要求查看竞品对比，应提供公开材料或授权后的简化版本，不能展示产品部内部分析原文。

## 5. 可回答的问题
- 竞品分析框架是什么？
- 产品部门内部知识库写的什么？
- 竞品分析有哪些维度？
- 竞品结论能不能直接对外传播？
- 竞品机会点如何进入需求池？
